Chapter 1: The Lay of the Land

To truly appreciate OopisOS, one must understand its foundational promise: to provide a complete, powerful, and persistent computing environment that respects the sanctity of your actual machine. Every file you create, every user account you register, every line of your command history is meticulously managed within the browser's own storage capabilities. This is accomplished through a clever combination of IndexedDB for the file system data and LocalStorage for user sessions and preferences, ensuring your digital world is waiting for you each time you return. This makes OopisOS the ultimate digital terrarium—a perfectly isolated sandbox where curiosity can run wild without any risk to your host system.

The primary interface is the command line, or "shell," which is your portal into this world. Its design is a deliberate nod to the legendary UNIX systems that form the backbone of modern computing, but with a modern, user-friendly implementation. The prompt itself is a dynamic status report, generated by the `TerminalUI.updatePrompt` function, that tells you everything you need to know at a glance. The `Guest@OopisOs:/>` format breaks down into `username@hostname:current_path>`, followed by the prompt character, which can even be configured in the `/etc/oopis.conf` file. It’s a small detail, but it’s part of a design philosophy that prioritizes clarity and control, inviting you to take command from your very first keystroke.

Chapter 2: The Core Lexicon - Navigating the World

While the "Holy Trinity" of `pwd`, `ls`, and `cd` are your primary tools for navigation, their implementation in OopisOS v2.6 reveals a deeper sophistication. The `pwd` command is a simple, direct call to `FileSystemManager.getCurrentPath()`, giving you an immediate and accurate sense of place. The `cd` command, however, is smarter than it looks; it understands that you can't change directory into a file, and more importantly, it respects the file system's permissions. If a directory doesn't have the "execute" bit set for your user, the `cd` command will politely refuse entry, enforcing the security model at a fundamental level.

The `ls` command has been truly supercharged in this version. Beyond the standard long format (`-l`) and showing all files (`-a`), it now includes a host of sorting options that allow you to organize your view of the file system precisely how you want. You can sort by modification time with `-t`, by file size with `-S`, alphabetically by file extension with `-X`, or even see items in their raw, unsorted directory order with `-U`. The long-listing format (`-l`) itself is a masterclass in information density, meticulously constructed by the `formatLongListItem` function to show you a file's type and permissions, its owner and group, its size, its last modification time, and its name—all in one clean line. It transforms a simple directory listing into a detailed system diagnostic.

The Holy Trinity of Navigation

  • pwd (Print Working Directory): The "You Are Here" map. It tells you your exact location in the file system's hierarchy.
  • ls (List): Shows you the contents of your current location. This is the most versatile tool for exploration, with powerful flags like -l for a detailed long-form view, -a to see hidden files (those starting with a '.'), -R to recursively list all subdirectories, and multiple sorting options (-t by time, -S by size, -X by extension, -U for unsorted).
  • cd [directory] (Change Directory): Your method of transport. Use it to move into other directories. `cd ..` takes you up one level.

Chapter 3: The Spark of Creation

Creation in OopisOS is a multi-faceted process, ranging from the instantaneous to the deeply considered. Simple commands like `mkdir -p` allow for the rapid scaffolding of complex project structures in a single line, while `touch` can create an empty file or simply update an existing one's timestamp. The combination of `echo` with the redirection operators (`>` and `>>`) provides a powerful, script-friendly method for creating and appending to files directly from the command line, as demonstrated in the screenshot. Furthermore, new analytical tools like `tree` give you a beautiful, top-down view of your creations, and `diff` lets you compare them with surgical precision.

The undisputed crown jewel of creation, however, is the `edit` command. This is far more than a simple text pad; it's a context-aware creative suite. The `EditorManager` intelligently determines the file mode—plain text, Markdown, or HTML—based on the file's extension. This transforms the user experience. For `.md` and `.html` files, a live preview pane appears, and a rich formatting toolbar provides one-click access to bold, italics, lists, and more. Productivity is further enhanced by a suite of keyboard shortcuts, including the essential `Ctrl+S` to save and exit, `Ctrl+O` to exit without saving (after a confirmation for unsaved work), and `Ctrl+P` to cycle through editor-only, preview-only, and split-view modes. This single command encapsulates the OopisOS philosophy: providing powerful, sophisticated tools in an accessible and self-contained package.

Chapter 4: The Power of the Shell

The OopisOS shell is where simple commands are forged into complex, powerful workflows. The `lexpar.js` module first tokenizes your input, then the `CommandExecutor`'s `_executePipeline` function masterfully orchestrates the flow of data. The pipe (`|`) operator is the heart of this system, taking the output of one command and seamlessly feeding it as the input to the next, allowing for on-the-fly filtering and processing. Redirection (`>` and `>>`) works in tandem, giving you ultimate control over where the final output lands, either by overwriting a file or appending to it.

Version 2.6 introduces two major enhancements to the shell's power: aliasing and environment variables. The `alias` command allows you to create your own shortcuts, which are resolved by the `AliasManager` before any other processing occurs. The new `set` and `unset` commands manage session-specific variables, which can be expanded in any command using `$VAR` syntax. Both aliases and environment variables are saved and persist across sessions, tailoring the OS to your personal workflow. Concurrently, the ampersand (`&`) operator allows you to run long processes in the background without tying up your terminal. These backgrounded jobs are not lost to the ether; they are tracked in an `activeJobs` list, which can be viewed with the `ps` command and terminated with the `kill` command, giving you full control over your multitasking environment.

  • Pipes |: The soul of the shell. The pipe takes the output of the command on its left and uses it as the input for the command on its right.
  • Redirection > and >>: Control where your output goes. > will send output to a file, overwriting its contents. >> will append the output to the end of a file.
  • Sequencing ;: String multiple commands together on a single line to be executed one after another.
  • Backgrounding &: Add an ampersand to the end of a command to run it in the background. Manage these jobs with ps (list) and kill (terminate).
  • Aliasing with alias: Create your own command shortcuts! For example, alias ll='ls -la' makes typing `ll` equivalent to `ls -la`.
  • Environment Variables with set and unset: Create temporary variables like set NAME="Chip" and use them later with echo "Hello, $NAME".

Guest@OopisOs:/> history | grep -c "edit"

Chapter 5: The System and the Self

OopisOS v2.6 elevates system management from a simple utility to a core feature, offering deep control over both your digital identity and the environment itself. The user and group system is a cornerstone of this design. When you `useradd` a new account, the system not only prompts for a password that is then securely hashed, but it also creates a primary group for that user and a private, protected home directory. This security model is built on a 3-digit octal permission system, familiar to any UNIX veteran. `chmod` is used to set read (4), write (2), and execute (1) permissions for the file's owner, its group, and all other users. This allows for the creation of sophisticated collaboration schemes, such as a shared project directory where members of the "developers" group can write files, while others can only read them.

This robust user model is complemented by a suite of powerful, system-aware utilities. The `find` command is a developer's dream, capable of scouring the entire file system for files matching specific criteria—from name and type to permissions and modification time—and then executing another command upon them with `-exec`. The `gemini` command brings artificial intelligence into the shell, but with a crucial twist: it can use the system's own tools (`ls`, `cat`, `tree`, etc.) to gather real-time data from your file system before formulating its response, providing answers of unparalleled relevance. Finally, for pure fun and a showcase of the system's versatility, the `adventure` and `paint` commands launch full creative applications in dedicated modal windows, proving that even in a serious operating system, there's always room for play.

A Multi-User Metropolis with Groups

You are not alone. With useradd, you can create new user accounts, each with a password-protected home directory. The login, logout, and su commands allow for seamless switching between users. But v2.6 introduces a full group system! Use groupadd, usermod, and chgrp to create and manage groups, controlling access to shared resources with the refined `chmod` command. The superuser, root (password: `mcgoopis`), has the ultimate authority.

The Unforgiving `find`

The `find` command is the system's bloodhound. It can search the entire file system based on a dizzying array of criteria: name (-name "*.txt"), type (-type d for directory), owner (-user Guest), permissions (-perm 755), and more. But its true power lies in its ability to execute actions on what it finds. You can print the results (-print), delete them (-delete), or even run another command on each result (-exec).

// Using `find` to locate all temporary files in the home directory and delete them.
root@OopisOs:/#> find /home -name "*.tmp" -delete

An AI in the Shell: `gemini`

OopisOS v2.6 integrates the Gemini AI directly into the terminal. But this is no simple chatbot. Ask it a question like gemini "summarize my project readme and list the scripts", and it can use the built-in commands (`ls`, `cat`, `find`, `tree`) to explore your file system, gather information, and provide a truly context-aware answer. It's like having a helpful digital librarian living inside your machine.

Creative Pursuits: `adventure` and `paint`

All work and no play makes for a dull OS. The adventure command launches a full-featured text adventure game. For the more visually inclined, the new `paint` command opens a character-based art studio, complete with a color palette and tools to create your next ASCII masterpiece. Both are testaments to the system's versatility and provide delightful digital distractions.

Total System Recall

Never fear a catastrophic error again. OopisOS provides a suite of commands for total system control. `savestate` and `loadstate` allow for manual session snapshots, while `backup` creates a downloadable JSON file of your entire OS state. Should you need to start over, `restore` can rebuild your world from that backup file, and the ultimate `reset` command will return the entire OS to its factory-fresh state (after a stern confirmation, of course).
